How It Works

// 1. Define lifecycle points
#[LifeCyclePoint('before_payment', ['user_id', 'amount'])]
#[LifeCyclePoint('after_payment', ['user_id', 'amount', 'payment_id'])]
class PaymentService
{
    use HasLifecycle;
    
    public function process(int $userId, float $amount): string
    {
        // Hooks can modify $amount by reference!
        runHook($this, 'before_payment', $userId, $amount);
        
        // Core logic uses potentially modified values
        $paymentId = $this->doPayment($userId, $amount);
        
        runHook($this, 'after_payment', $userId, $amount, $paymentId);
        
        return $paymentId;
    }
}

Creating Hooks

// Critical severity hook
#[Hook(
    scope: 'PaymentService', 
    point: 'before_payment', 
    severity: Severity::Critical
)]
class FraudDetectionHook
{
    use Hookable;
    
    public function handle(array &$args): void
    {
        if ($this->isFraudulent($args)) {
            throw new FraudException('Suspicious activity detected');
        }
    }
}

// Optional hook that modifies data
#[Hook(scope: 'PaymentService', point: 'before_payment', severity: Severity::Optional)]
class ApplyDiscountHook
{
    public function handle(array &$args): void
    {
        // Modifies amount by reference!
        $args['amount'] *= 0.9; // 10% discount
    }
}

Reference Mutability

Hooks can chain-modify data

// Initial value
$amount = 100.00;
$userId = 123;

// Execute hooks registered in Kernel:
// 1. ApplyDiscountHook    ‚Üí $amount = 90.00  (10% discount)
// 2. ApplyTaxHook         ‚Üí $amount = 97.20  (8% tax)
// 3. RoundingHook         ‚Üí $amount = 97.00  (round down)

runHook(PaymentService::class, 'before_payment', $userId, $amount);

echo $amount; // 97.00 - directly modified by hooks!
  • Variables are automatically passed by reference
  • Multiple hooks can modify the same data
  • Execution order matters (defined in Kernel)
  • Framework automatically maps variables to array

Centralized Control

app/Hooks/Kernel.php

class Kernel
{
    public array $hooks = [
        \App\Services\PaymentService::class => [
            'before_payment' => [
                \App\Hooks\ValidateAmountHook::class,     // executes 1st
                \App\Hooks\FraudDetectionHook::class,     // executes 2nd
                \App\Hooks\ApplyDiscountHook::class,      // executes 3rd
            ],
            'after_payment' => [
                \App\Hooks\SendEmailHook::class,
                \App\Hooks\UpdateAnalyticsHook::class,
            ]
        ]
    ];
}
  • Explicit and controlled execution order
  • Complete visibility of all extensions
  • Easy for code review and debugging
